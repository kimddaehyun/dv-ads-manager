# 디브이 애드 매니저 MVP PRD

## 🎯 핵심 정보

**목적**: 네이버 광고 대시보드(`ads.naver.com`)에서 파워링크 키워드의 순위/예상 입찰가와 쇼핑검색광고 소재의 자동매칭 키워드별 현재 순위/예상 입찰가를 실시간으로 옆에 띄워 입찰 의사결정 시간을 단축한다.
**사용자**: 다수의 광고주 계정을 동시에 운영하는 **네이버 광고 대행사 AE**가 1차 타겟. 인하우스 광고 운영자, 브랜드 스토어 셀러도 포함.

> 본 프로젝트는 Chrome MV3 확장으로 배포된다. 일반 웹 앱이 아니라 호스트 페이지(`ads.naver.com`)에 콘텐츠 스크립트를 주입하는 구조이며, 회원 인증 대신 **라이선스 키 검증**(Supabase RPC) 방식을 사용한다. 검색광고 API의 입찰가/볼륨 응답은 **시장 단위 추정치**이므로 호출자 customerId와 무관하게 동일하다 — 따라서 자격증명은 **`customerId` + `accessLicense` + `secretKey` 1쌍만 등록**해 두면 사용자가 보고 있는 광고주가 누구든 동일하게 동작한다. 광고주 매칭·다중 자격증명 개념은 사용하지 않는다. 코어 라이브러리는 자매 프로젝트 `naver-tag-picker`와 공유한다.
>
> **🟡 TBD**: 쇼핑검색광고 키워드별 순위/예상 입찰가의 **데이터 소스는 미정**(검색광고 API 쇼핑 endpoint / `ads.naver.com` DOM 추출 / 내부 XHR 재호출 등). 자매 프로젝트의 `src/lib/search-popular.ts`는 스마트스토어 셀러 백오피스 API(`sell.smartstore.naver.com`)를 호출하지만, 대행사 AE 타겟에서는 셀러 계정을 보유하지 않아 부적합. 구현 단계에서 별도 결정.

## 🚶 사용자 여정

```
1. Chrome 웹스토어에서 확장 설치
   ↓ 설치 직후
2. 옵션 페이지 (자동 또는 사용자 직접 진입)
   ↓ 라이선스 키 입력 → Supabase RPC 검증 (베이직 tier 활성 시 전체 기능 해제)
   ↓ 검색광고 API 자격증명(customerId + accessLicense + secretKey) 1쌍 등록
3. ads.naver.com 광고 대시보드 방문
   ↓ 콘텐츠 스크립트가 URL 패턴에 따라 화면 종류 감지
   [자격증명 등록] → 파워링크 오버레이 활성 (시장 단위 입찰가/볼륨)
   [자격증명 미등록] → "검색광고 API 자격증명 미등록" 안내 + 옵션 페이지 바로가기
   ↓ URL 패턴에 따라 분기
   [파워링크 키워드 화면]  → 콘텐츠 오버레이 — 파워링크 키워드 테이블
   [쇼핑검색광고 그룹 화면] → 콘텐츠 오버레이 — 쇼핑검색광고 그룹 뷰
   [쇼핑검색광고 소재 화면] → 콘텐츠 오버레이 — 쇼핑검색광고 소재 상세
4. (필요 시) 팝업 페이지 진입 → 라이선스 상태 확인, 캐시 강제 갱신
5. (자격증명 수정 필요 시) 옵션 페이지 재진입
```

## ⚡ 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|---------------|-------------|
| **F001** | 파워링크 키워드 순위·입찰가 오버레이 | 파워링크 키워드 테이블 각 행 옆에 현재 세팅된 입찰가의 추정 순위(예: "현재 4위")와 1위~10위의 예상 입찰가를 표시. 검색광고 API 자격증명이 등록돼 있을 때만 활성. **데이터 소스**: 네이버 검색광고 API `POST /estimate/average-position-bid/keyword` (요청 body의 `items: [{key, position}]`로 키워드별 1~10위 입찰가 일괄 조회). 응답은 시장 단위 추정치라 어떤 자격증명으로 호출해도 같은 숫자가 나옴 — 광고주 매칭 불필요. 응답 schema의 정확한 필드명은 구현 1일차 실호출로 확정 | 본 확장의 첫 번째 핵심 가치. 입찰가 결정 시 광고 대시보드를 떠나지 않고 즉시 비교 가능 | 콘텐츠 오버레이 — 파워링크 키워드 테이블 |
| **F002** | 쇼핑검색광고 그룹 뷰 소재별 inline 펼침 | 그룹 페이지의 소재 행 옆 토글 버튼을 누르면 그 자리에 펼쳐지며, 소재의 자동매칭 키워드 목록과 각 키워드의 현재 순위·1위~10위 예상 입찰가를 테이블로 표시. **여러 소재를 동시에 펼쳐 비교 가능**. **데이터 소스는 TBD** (위 핵심 정보 참고) | 두 번째 핵심 가치. 쇼핑검색광고는 키워드를 직접 지정할 수 없어 소재별 매칭 키워드 파악 자체가 운영 난점 | 콘텐츠 오버레이 — 쇼핑검색광고 그룹 뷰 |
| **F003** | 쇼핑검색광고 소재 상세 키워드 패널 | 소재 단독 상세 페이지에서 그 소재의 자동매칭 키워드 전체 목록과 키워드별 현재 순위·1위~10위 예상 입찰가를 풀 패널로 표시. 정렬/검색 가능. **데이터 소스는 TBD** | 그룹 뷰의 inline 펼침과 동일 정보를 단일 소재 페이지에서는 더 넓은 영역으로 보여줘야 정독 가능 | 콘텐츠 오버레이 — 쇼핑검색광고 소재 상세 |

### 2. MVP 필수 지원 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|---------------|-------------|
| **F010** | 라이선스 키 등록·검증 | 사용자가 입력한 라이선스 키를 Supabase RPC `verify_access`로 검증, 결과(tier·만료일)를 5분 TTL로 캐시. MVP에서는 **베이직 tier 단일 등급**이며 활성이면 모든 핵심 기능 사용 가능. 미검증·만료 시 모든 핵심 기능 비활성. 디바이스 ID와 키만 외부 전송 | 회원가입/로그인을 대체하는 본 확장의 유일한 인증·과금 게이트 | 옵션 페이지 |
| **F011** | 검색광고 API 자격증명 등록 | `customerId` + `accessLicense` + `secretKey` 1쌍을 등록·수정·삭제. 비밀값은 `chrome.storage.local`에만 저장되며 외부 전송 X. 검색광고 API 응답이 시장 단위 추정치라 광고주가 누구든 같은 데이터가 나오므로 자격증명은 1개면 충분 — 본 repo 어딘가에서 광고주 매칭이나 다중 자격증명 관리는 하지 않는다. 자격증명 자체의 storage 키(`searchadCredentials`)는 자매 프로젝트 `naver-tag-picker`와 공유되는 코어 라이브러리(`src/lib/searchad.ts`)가 관리한다 — 코드 변경 없이 그대로 사용 | 본 확장의 모든 검색광고 API 기능(F001 및 추후 F002/F003 일부)을 활성화하는 게이트. 자격증명 미등록 시 콘텐츠 오버레이에서 "자격증명 미등록" 안내 + 옵션 페이지 바로가기로 폴백 | 옵션 페이지 |
| **F012** | 라이선스 상태 표시 및 캐시 강제 갱신 | 팝업에서 라이선스 tier·만료일·검증 시각을 확인. "지금 다시 조회" 버튼은 활성 탭에 해당하는 키워드 캐시를 만료시키고 콘텐츠 스크립트에 재조회 트리거 메시지 전송 (전체 캐시 클리어 X). 활성 탭 식별은 `chrome.tabs.query({active:true, currentWindow:true})` 사용 — host_permissions만으로 충분한지 구현 첫날 1회 확인, 필요 시 `permissions`에 `"activeTab"` 추가 | 캐시가 오래된 값을 보여줄 때 사용자가 즉시 새 데이터로 갱신할 수 있어야 함 | 팝업 페이지 |

### 3. MVP 이후 기능 (제외)

- 키워드별 입찰가 변동 히스토리·차트
- 경쟁사 광고 분석
- 자동 입찰가 제안·일괄 변경
- 광고주별 일·주·월 단위 통계 요약 대시보드
- 슬랙·이메일 알림
- 다크 모드 / 테마 설정
- 데이터 내보내기 (CSV/엑셀)
- 라이선스 다중 tier(프로/엔터프라이즈) 및 tier별 기능 게이트
- 자격증명 가져오기/내보내기(JSON), 팀 단위 자격증명 공유

## 📄 페이지별 상세 기능

### 콘텐츠 오버레이 — 파워링크 키워드 테이블

> **구현 기능:** `F001` | **UI 위치:** `ads.naver.com` 파워링크 키워드 화면에 자동 주입

| 항목 | 내용 |
|------|------|
| **역할** | 파워링크 키워드 운영 화면에서 입찰가 결정에 필요한 순위·예상 입찰가 정보를 키워드 행 바로 옆에 inline 표시 |
| **진입 경로** | 사용자가 `ads.naver.com`의 파워링크 키워드 관리 화면에 진입하면 콘텐츠 스크립트가 URL 패턴을 감지하여 자동 활성화 |
| **사용자 행동** | 키워드 행을 훑으며 "현재 입찰가는 몇 위인지", "1위가 되려면 얼마를 더 써야 하는지"를 즉시 비교. 자격증명이 미등록이면 옵션 페이지에서 등록하고 돌아옴 |
| **주요 기능** | • 키워드 행 옆에 **현재 입찰가의 추정 순위** 배지 표시 (예: "현재 4위")<br>• 행 펼침 시 **1위~10위의 예상 입찰가** 테이블 표시<br>• 자격증명 미등록 시 "검색광고 API 자격증명 미등록" 안내 배지 + 옵션 페이지 바로가기<br>• 라이선스 미검증/만료 시 잠금 상태 표시<br>• 검색광고 API 429/401/네트워크 에러 시 친화적 메시지 표시 |
| **다음 이동** | 동일 페이지 내 키워드 추가/수정/삭제 시 오버레이 자동 갱신. 페이지 이탈 시 콘텐츠 스크립트 정리. 미등록 안내 클릭 시 옵션 페이지 |

| 항목 | 내용 |
|------|------|

### 콘텐츠 오버레이 — 쇼핑검색광고 소재 상세

> **구현 기능:** `F003` | **UI 위치:** `ads.naver.com` 쇼핑검색광고 소재 상세 화면에 자동 주입

| 항목 | 내용 |
|------|------|
| **역할** | 단일 소재에 집중해 자동매칭 키워드 목록·현재 순위·순위별 예상 입찰가를 넓은 영역에서 정독할 수 있게 한다. 그룹 뷰의 압축된 inline 표보다 더 많은 컬럼·정렬·필터를 제공 |

## 🗄️ 데이터 모델

본 확장은 서버 DB를 보유하지 않는다. 사용자 측 영속 저장소는 `chrome.storage.local`이며, 라이선스 검증만 Supabase의 기존 RPC를 호출한다.

### LicenseState (chrome.storage.local 키: `license`)
| 필드 | 설명 | 타입 |
|------|------|------|
| key | 사용자가 입력한 라이선스 키 원문 | string |
| device_id | 본 확장이 디바이스를 식별하기 위해 생성한 UUID | string |
| tier | Supabase RPC가 반환한 등급 (MVP는 `basic` 단일) | string |
| expires_at | 라이선스 만료 일시 | ISO date string |
| verified_at | 마지막 검증 성공 시각 | ISO date string |

### SearchadCredentials (chrome.storage.local 키: `searchadCredentials`)
> 자격증명 1쌍. 코어 라이브러리 `src/lib/searchad.ts`가 관리하므로 본 repo는 키 형태를 그대로 사용한다 (자매 프로젝트 `naver-tag-picker`와 동기화).

| 필드 | 설명 | 타입 |
|------|------|------|
| customerId | 네이버 검색광고 광고주 ID (HMAC `X-Customer` 헤더용) | string |
| accessLicense | API 액세스 라이선스 | string |
| secretKey | API 시크릿 키 (HMAC 서명용) | string |

### KeywordVolumeCache (chrome.storage.local 키: `volume_cache:<keyword>`)
> 검색광고 API 응답이 시장 단위 추정치라 호출자 customerId와 무관하게 동일 — 키워드 단위로만 스코프.

| 필드 | 설명 | 타입 |
|------|------|------|
| keyword | 정규화된 키워드 | string |
| rank_to_bid | 순위(1~10) → 예상 입찰가 매핑 | object |
| fetched_at | 캐시 적재 시각 | ISO date string |

### ShoppingRankCache (chrome.storage.local 키: `shopping_cache:<product_id>:<keyword>`)
| 필드 | 설명 | 타입 |
|------|------|------|
| product_id | 쇼핑 소재 식별자 | string |
| keyword | 자동매칭 키워드 | string |
| rank | 현재 노출 순위 (미노출이면 null) | number \| null |
| fetched_at | 캐시 적재 시각 | ISO date string |

### CurrentBidSnapshot (chrome.storage.local 키: `current_bid:<keyword>`)
| 필드 | 설명 | 타입 |
|------|------|------|
| keyword | 정규화된 키워드 | string |
| current_bid | 광고 대시보드에서 읽어낸 현재 입찰가 | number |
| estimated_rank | rank_to_bid 맵을 기준으로 추정한 현재 순위 | number |
| read_at | DOM에서 읽어낸 시각 | ISO date string |

- `src/lib/searchad.ts`·`license.ts`·`supabase.ts`·`friendly-error.ts`는 자매 프로젝트와 1:1 동기화 유지가 원칙(CLAUDE.md "코어 코드 변경 정책").
